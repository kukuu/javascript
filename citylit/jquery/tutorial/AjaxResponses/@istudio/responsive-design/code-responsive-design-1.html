                               <h2>Responsive Design & Mobile First</h2>   
                               <p>Responsive Design and Mobile First approach to web, mobile and wide screen program development (including smart TVs)
                                allows you to future proof your API in the fast changing market. In short, growing business without "tears".
                                 A flexible layout is better prepared for devices that haven’t even launched yet.</p> 

                                <p>Not long ago ago, mentioning “tablet” would conjure up images of the iPad in the listener’s mind.
                                 But now, seven inch tablets like Samsung’s Galaxy Tab are in use, and devices like the Kindle and Nook are also carrying fantastic little browsers. More are yet to come.
                                As developers, We simply can’t keep up coding the different resolutions entering the marketplace each time. We need a Design Pattern. 
                                One for resolving common occurrent issues, including the future. That means "assault proofing" your code!</p>

                                <p>A flexible foundation allows us to step back from targeting individual resolutions, and better prepare our designs for devices that haven’t even been imagined yet.</p>

                                <h3>Ingredients for Reponsive Design & Mobile First</h3>

                                <p>Using a simple relationship between objects and elements of the design in question, we can implement responsive design.</p>

                                <p>Formula: <i>target / context = result</i><br />
                                    where:<br />
                                    <i>target</i>: Element or object  being targeted<br />
                                    <i>context</i>: Parent element or object of the targeted element
                                </p>

                                <p>It takes three core ingredients to create a responsive design:</p>

                                <ul>
                                    <li>A flexible, grid-based layout</li>
                                    <li>Flexible images and media</li>
                                    <li>Media queries & Media Query Listeners (A module from the CSS3 specification).</li>                 
                                </ul>

                                <p>Click this <a href="css/responsive-design-1.CSS">  link</a> to see the responsive stylesheet for this work and the <a href="file:///Z:/DROPBOX/ALEXANDER%20ADU-SARKODIE/snippets/responsive-design/code-responsive-design-1.html">source code</a> for the page. 
                                This document has been written in tutorial format. So some  properties of certain Classes and IDs (previously defined) may be re-defined to overide previous set values.</p>

                                <p>Note whiles developing, you can simply resize the browser window to a few different widths. However, this is no substitute 
                                for actually testing your work on separate devices. But it allows to quickly assess how the design handles several different resolution ranges, 
                                and simulate how users on capable phones, tablets, or other devices might experience our design. 
                                It is recommended to have a test suite to cope with the development cycle.</p>

                                <h2  class="clear-left">Less CSS</h2>  
                                <p> LESS CSS is a  preprocessor language that allows you to write less CSS code. It extends CSS with dynamic behavior such as variables,
                                    mixins, operations and functions. </p>

                                <p>During translation, the values of the variables, mixins, operations and functions are inserted into the output CSS document.</p>

                                <p>This work on  LESS CSS looks at high level, organisational structure and dependencies  between the stylesheet that aggregate to produce the page.</p>

                                    <li>components</li>

                                </ul>
