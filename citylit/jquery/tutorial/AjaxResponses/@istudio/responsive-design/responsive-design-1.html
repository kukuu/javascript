Responsive Design & Mobile First

Responsive design and Mobile First approach to web, mobile and wide screen program development (including smart TVs) allows you to future proof your API in the fast changing market. In short, growing business without "tears". A flexible layout is better prepared for devices that haven’t even launched yet.

Not long ago ago, mentioning “tablet” would conjure up images of the iPad in the listener’s mind. But now, seven inch tablets like Samsung’s Galaxy Tab are in use, and devices like the Kindle and Nook are also carrying fantastic little browsers. More are yet to come. As developers, We simply can’t keep up coding the different resolutions entering the marketplace each time. We need a Design Pattern. One for resolving common occurrent issues, including the future. That means "assault proofing" your code!

A flexible foundation allows us to step back from targeting individual resolutions, and better prepare our designs for devices that haven’t even been imagined yet.

Ingredients for Reponsive Design & Mobile First

Using a simple relationship between objects and elements of the design in question, we can implement responsive design.

Formula: target / context = result
where:
target: Element or object being targeted
context: Parent element or object of the targeted element

It takes three core ingredients to create a responsive design:

  • A flexible, grid-based layout
  • Flexible images and media
  • Media queries & Media Query Listeners (A module from the CSS3 specification).

Click this link to see the responsive stylesheet for this work. This document has been written in tutorial format. So some properties of certain Classes and IDs (previously defined) may be re-defined to overide previous set values.

Note whiles developing, you can simply resize the browser window to a few different widths. However, this is no substitute for actually testing your work on separate devices. But it allows to quickly assess how the design handles several different resolution ranges, and simulate how users on capable phones, tablets, or other devices might experience our design. It is recommended to have a test suite to cope with the development cycle.

Less CSS

LESS CSS is a preprocessor language that allows you to write less CSS code. It extends CSS with dynamic behavior such as variables, mixins, operations and functions.

During translation, the values of the variables, mixins, operations and functions are inserted into the output CSS document.

This work on LESS CSS looks at high level, organisational structure and dependencies between the stylesheet that aggregate to produce the page.

Directory where files are stored:
/less-css/rnd/css

In the parent CSS folder, there is this organisation of folders:

  • base
  • components
  • styleguide
  • views

And in the base folder for example, there are these less files:

  • compact.less
  • core.less
  • tablet.less
  • wide.less

For example, core.less imports style from:

  • styleguide folder (i.e. reset.less and typography.less)
  • layout grid styles from the view folder (views/core.less)
  • Individual components less files, in the three main sections of the page
    • Banner
    • Articles (left and right sections of main content area)
    • Aside (right section of main content area)

There is some computation in the following files

  • /less-css/rnd/css/base/core.less (core.less)
    This LESS file will import, compile, strip comments and flattens all LESS styles into a single CSS for core devices;
    • feature phones
    • non-js
    • older browsers
  • /less-css/rnd/css/base/wide.less (wide.less)
    Flattens all LESS styles into a single CSS for desktop browsers;
    • modern browsers that 'cut the mustard'
    • support media queries and media query listeners
    • screen is wider than 1200px, browser window is more than 1056px wide (Including support for smart TVs)
  • /less-css/rnd/css/styleguide/reset.less (reset.less)
    Based on Eric Meyer's reset.
  • /less-css/rnd/css/styleguide/typography.less (typography.less)
